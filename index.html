k3c · onboarding

Real k3s clusters on your Mac. No Docker Desktop.

k3c runs Kubernetes and a real Docker engine as lightweight VMs on Apple's native container runtime — and takes care of everything that usually hurts on a corporate laptop: VPN egress, CA bundles, registry mirrors, memory, and instant snapshots. Free and open source: no subscription, no per-seat license, nothing to procure.

zsh — ~/dev

macOS / Apple silicon · the Apple container runtime ships inside the binary — brew install and go

What k3c does for you

One binary, one YAML file per project. Everything below works out of the box or with one config line.

No license. No subscription.

k3s on Apple's Virtualization.framework — free and open source, no Docker Desktop seat to buy. One VM per machine, Rosetta for amd64 images, kubeconfig handled automatically.

k3c cluster create

Corporate VPN? Handled.

Transparent egress (default): every VM connection is re-originated from host sockets, so Zscaler/VPN, proxies, and corporate DNS just work — no per-domain config.

k3c doctor verifies the whole chain

A real Docker engine

The docker:dind sidecar exposes a genuine Engine API. Testcontainers, docker CLI, and buildx work unchanged — docker up activates the context for you.

k3c docker up

Instant snapshots

APFS copy-on-write snapshots of the whole cluster — including RAM. Break things freely; restore in seconds. Export them to teammates.

k3c snapshot save

Your RAM stays yours

Memory ballooning returns unused guest RAM to macOS continuously; pause or suspend clusters you're not using and switch in under a second.

k3c cluster reclaim · pause · suspend

Prod charts on a laptop

An admission webhook rewrites pod CPU/memory requests to near-zero (limits stay), so production-sized Helm charts schedule locally.

ignoreCpuRequests: true

Fast image pulls, shared

A pull-through cache serves every cluster; registry mirrors and private CAs from your k3c.yaml are baked into nodes, buildkit, and k3d images.

k3c image pull-cache list

See everything live

A terminal UI for machines + snapshots and a browser diagram with live traffic, pod heatmaps, and one-click lifecycle actions.

k3c ui · k3c web

Snapshots come in three temperatures

Pick the tier by what you want back and how fast. All three are APFS clones — saving is instant.

warm

default · restores instantly

Disk and RAM. Restore resumes the running machine mid-thought — pods, connections, everything. Largest on disk.

k3c snapshot save

cold

boots fresh · ~30–60 s

Full disk, no machine state. IP-independent and smaller; the cluster boots clean from the saved state.

k3c snapshot save --cold

frozen

smallest · thaws in minutes

Just the datastore, volumes, and an image manifest — images rehydrate from the pull-cache on restore. Ideal for sharing.

k3c snapshot save --frozen

Command cheat sheet

Cluster

k3c cluster create [NAME] — create & start

k3c cluster list — all clusters, current marked

k3c cluster activate NAME — switch (<1 s with pause)

k3c cluster stop · start · delete

k3c status — cluster, daemons, nodes

Memory & speed

k3c cluster pause / resume — freeze in RAM, instant

k3c cluster suspend — to disk, frees CPU+RAM

k3c cluster reclaim [--release] — balloon RAM back

Snapshots

k3c snapshot save [NAME] [--cold|--frozen]

k3c snapshot restore NAME [--cold]

k3c snapshot export NAME -o f.k3csnap

k3c cluster import-run f.k3csnap — clone a teammate's

Docker sidecar

k3c docker up — engine + context, ready for Testcontainers

k3c docker env — exports for CI shells

k3c docker buildkit — buildx behind the corporate CA

k3c docker snapshot save/restore — whole image store

Images

k3c image import IMAGE — host store → cluster

k3c image pull-cache list · info · prune

When something's off

k3c doctor — walks every known failure mode, read-only

k3c doctor --shell — netshoot pod inside the cluster

k3c daemons restart — refresh host listeners

k3c info — version, runtime, config in use
